# Café Apokalypso – Art Styleguide

## Core direction
Small, dense, cozy-absurd pixel café in a light 3/4 diorama perspective.

## Main view
- 3/4 diorama perspective
- no side-view as default gameplay
- compact café interior
- readable counters, tables, guests, machines and UI zones
- cozy first impression, weird details only gradually visible

## Mood
- warm
- charming
- lightly absurd
- not horror
- not grimdark
- not chaotic meme collage

## Pixel style
- pixel-art inspired
- readable at small sizes
- limited detail density per object
- no glossy AI-render look
- no pseudo-realistic illustration
- no generic mobile-game asset look

## Color direction
- warm café base palette
- creams, browns, muted reds, soft yellows
- weirdness can introduce cold cyan, violet, sickly green, bureaucratic grey
- apocalypse elements should contrast with cozy café elements

## What Codex may do
- create CSS-based placeholders
- arrange asset folders
- reference placeholder assets
- add typed asset metadata
- implement rendering components
- keep naming conventions consistent

## What Codex may not do without approval
- generate final art
- add external image assets
- introduce a different view style
- create side-view main gameplay assets
- add large SVG illustrations as final art
- change the color direction
- invent new character designs as canonical
- commit AI-generated images without review

## Placeholder rule
Placeholders are allowed if they are clearly marked as placeholders and easy to replace later.

Recommended naming:
- `placeholder-cafe-diorama`
- `placeholder-guest-normal-01`
- `placeholder-guest-strange-01`
- `placeholder-kassandra-ui`
